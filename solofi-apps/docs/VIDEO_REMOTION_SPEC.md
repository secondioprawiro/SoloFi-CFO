# SoloFi CFO — Remotion Build Spec

Addendum to `docs/VIDEO_PITCH_SCRIPT.md`. That file has narration/timing; this
file has what Remotion actually needs to render: scenes, assets, durations,
visual spec. Read both before scaffolding.

**Hard constraint (OKX.AI Genesis Hackathon rules, Step 3):** the X post demo
must be **no longer than 90 seconds**. This is not a target range — it's a
submission-validity requirement. Everything below is cut to fit exactly 90s;
tech-stack and roadmap scenes from the earlier 3:00 draft are dropped
entirely — there's no time budget for them and the rules only require
"introduce ASP, explain use case, demo/walkthrough," not a full pitch deck.

## Project setup

- Separate folder: `video/` at repo root — own `package.json`, does not touch
  the backend build (`src/`, `tsconfig.json`).
- Scaffold with the Remotion CLI (from repo root):
  ```bash
  npx create-video@latest --yes --blank --no-tailwind video
  cd video
  npx remotion studio
  ```
  `--blank` skips demo boilerplate, `--no-tailwind` since this spec's styling
  is plain CSS-in-JS (Tailwind animation classes don't render correctly in
  Remotion anyway — see rule below). `npx remotion studio` opens the live
  preview at `localhost:3000`, hot-reloads on save — use this to iterate on
  scenes instead of re-rendering the full MP4 every time.
- Composition lives in `src/Root.tsx`:
  ```tsx
  import { Composition } from "remotion";
  import { MainComposition } from "./MainComposition";

  export const RemotionRoot = () => (
    <Composition
      id="MainComposition"
      component={MainComposition}
      durationInFrames={2700}
      fps={30}
      width={1920}
      height={1080}
    />
  );
  ```
- Resolution: **1920×1080**, **30fps**. Total: **1:30 = 2700 frames**.
- **Animation rule:** no CSS transitions/animations, no Tailwind animation
  classes — Remotion renders frame-by-frame and both are silently ignored.
  Animate with `useCurrentFrame()` + `interpolate()`:
  ```tsx
  import { useCurrentFrame, useVideoConfig, interpolate, Easing } from "remotion";

  const opacity = interpolate(useCurrentFrame(), [0, 30], [0, 1], {
    extrapolateLeft: "clamp",
    extrapolateRight: "clamp",
    easing: Easing.bezier(0.16, 1, 0.3, 1),
  });
  ```
- One-frame sanity check without a full render (e.g. to check scene-2's
  first frame lines up with the terminal recording):
  ```bash
  npx remotion still MainComposition --scale=0.25 --frame=600
  ```
  (`--frame` is zero-based; `600` = the 0:20 mark where scene 2 starts.)

## Required assets (must exist before scaffolding — not generated by Remotion)

### 1. Voice (narration audio)

- **What:** one continuous VO track, ~90s spoken, matching the trimmed script
  in `VIDEO_PITCH_SCRIPT.md` (~200–220 words total at natural pace — script
  must be word-counted before recording, not after, so it doesn't overrun).
- **File:** `video/public/voiceover.mp3`.

**Option A — record your own voice (recommended: free, no API cost, matches
this project's free-tier-only constraint):**
- **Windows Voice Recorder** (built into Windows 11, `Win` → search "Voice
  Recorder") — zero setup, outputs `.m4a`, good enough for a hackathon demo.
- **Audacity** (free, [audacityteam.org](https://www.audacityteam.org)) — if
  you want basic noise reduction / trimming before export: record, then
  `Effect → Noise Reduction`, `File → Export → Export as MP3`.
- Read the script from `VIDEO_PITCH_SCRIPT.md` at natural pace, time it —
  if over ~90s, trim wording (not speaking speed) and re-record.

**Option B — TTS if no one wants to record (ElevenLabs, free tier):**
```ts
// video/scripts/generate-voiceover.ts
const response = await fetch(
  `https://api.elevenlabs.io/v1/text-to-speech/${voiceId}`,
  {
    method: "POST",
    headers: {
      "xi-api-key": process.env.ELEVENLABS_API_KEY!,
      "Content-Type": "application/json",
      Accept: "audio/mpeg",
    },
    body: JSON.stringify({
      text: "Meet SoloFi CFO — an autonomous finance agent on OKX.AI...", // full VO script
      model_id: "eleven_multilingual_v2",
      voice_settings: { stability: 0.5, similarity_boost: 0.75, style: 0.3 },
    }),
  },
);
require("fs").writeFileSync(
  "video/public/voiceover.mp3",
  Buffer.from(await response.arrayBuffer()),
);
```
```bash
node --strip-types video/scripts/generate-voiceover.ts
```
Check ElevenLabs free-tier character limits before committing to this path —
no paid key, same rule as the Gemini fallback work elsewhere in this project.

**Enforcing the 90s cap automatically:** instead of guessing frame numbers by
hand, measure the actual recorded VO length and let Remotion size the
composition to it via `calculateMetadata` — this also catches an overrun
before you render:
```tsx
// video/src/Root.tsx
import { CalculateMetadataFunction, staticFile } from "remotion";
import { getAudioDurationInSeconds } from "@remotion/media-utils";

const calculateMetadata: CalculateMetadataFunction<{}> = async () => {
  const seconds = await getAudioDurationInSeconds(staticFile("voiceover.mp3"));
  if (seconds > 90) {
    throw new Error(`VO is ${seconds.toFixed(1)}s — over the 90s hackathon cap. Trim the script and re-record.`);
  }
  return { durationInFrames: Math.round(seconds * 30) };
};
```

### 2. Screen recording (the demo)

- **What:** `video/public/x402-demo.mp4` — real terminal run of:
  ```bash
  npx tsx scripts/test-x402-payment.ts
  ```
  Full-screen terminal, readable font (≥18pt). Must show all three beats:
  `402` challenge → signed `X-PAYMENT` retry → `200 OK` with invoice created.
  Real run, not faked — the script already works end-to-end.

**Recording tool (pick one):**
- **Windows Game Bar** (built in, `Win+G`) — zero install, zero cost. Open a
  full-screen terminal, `Win+G` → record button (or `Win+Alt+R` to start/stop
  directly). Saves to `Videos/Captures/`. Good enough for a terminal-only
  capture; no cursor/webcam overlay needed here.
- **OBS Studio** (free, [obsproject.com](https://obsproject.com)) — if you
  want more control (crop to just the terminal window, higher bitrate, no
  Windows watermark edge case). Add a "Window Capture" source targeting the
  terminal, set output to 1920×1080 to match the composition and skip a
  resize step later.
- **ShareX** (free, [getsharex.com](https://getsharex.com)) — lighter-weight
  alternative to OBS if you just need a quick screen-region recording.

**Trimming the recording to fit the ~55s scene-2 window** — record the full
run, don't try to force the live run to exactly 55s. Remotion ships its own
ffmpeg, no separate install needed:
```bash
npx remotion ffprobe video/public/x402-demo-raw.mp4        # check actual duration first
npx remotion ffmpeg -ss 00:00:03 -i video/public/x402-demo-raw.mp4 \
  -to 00:00:58 -c:v libx264 -c:a aac video/public/x402-demo.mp4
```
(`-ss`/`-to` re-encodes from the exact frame so there's no frozen start
frame — required when trimming, not optional.) Alternatively use the
non-destructive `trimBefore`/`trimAfter` props directly on the `<Video>`
component in the composition instead of pre-trimming the file — preferred
if you're still iterating on the exact cut points:
```tsx
import { Video } from "@remotion/media";
import { staticFile } from "remotion";

<Video src={staticFile("x402-demo.mp4")} trimBefore={3 * 30} trimAfter={58 * 30} />
```

Optional b-roll if time allows within the same 55s: 3–5s cut to the X Layer
block explorer showing the resulting on-chain transfer. Skip it if it pushes
scene 2 over budget — the terminal run alone satisfies "demo/walkthrough."

### 4. Captions (required — burn in, don't rely on X's auto-captions)

Two ways to get caption text + timing, pick based on whether VO is recorded
(Option A) or TTS (Option B) above:

**If TTS (ElevenLabs):** you already have the exact script text and can
hand-time it per scene — no transcription needed, just split
`VIDEO_PITCH_SCRIPT.md`'s VO lines across the scene frame ranges below.

**If recorded voice:** auto-transcribe with Whisper so captions actually
match what was said (word-count target ≠ what came out of your mouth):
```bash
npx remotion add @remotion/install-whisper-cpp
```
```ts
// video/scripts/transcribe.ts
import path from "path";
import { installWhisperCpp, downloadWhisperModel, transcribe, toCaptions } from "@remotion/install-whisper-cpp";
import fs from "fs";

const to = path.join(process.cwd(), "whisper.cpp");
await installWhisperCpp({ to, version: "1.5.5" });
await downloadWhisperModel({ model: "medium.en", folder: to });

// Whisper needs 16kHz wav — convert first via Remotion's bundled ffmpeg:
// npx remotion ffmpeg -i video/public/voiceover.mp3 -ar 16000 video/public/voiceover.wav

const output = await transcribe({
  model: "medium.en",
  whisperPath: to,
  whisperCppVersion: "1.5.5",
  inputPath: "video/public/voiceover.wav",
  tokenLevelTimestamps: true,
});
const { captions } = toCaptions({ whisperCppOutput: output });
fs.writeFileSync("video/public/captions.json", JSON.stringify(captions, null, 2));
```

**Rendering captions in the composition** — burn in as `<Sequence>` text
overlays, bottom-third, gold text on translucent navy bar (same palette as
above), timed either from `captions.json` (Whisper path) or hand-split VO
lines (TTS path), synced to the scene frame ranges from the table above.

## Rendering

```bash
npx remotion render MainComposition out/solofi-cfo-pitch.mp4
```
Local render, no API/token cost — safe to iterate on this step freely once
the composition code is written. Confirm final render duration is **≤ 90s**
before posting — the `calculateMetadata` check above already throws if the
VO alone overruns, but double check the rendered file itself too:
```bash
npx remotion ffprobe out/solofi-cfo-pitch.mp4
```
Re-export trimmed if it overruns — going over is a hard submission-invalidity
risk per the hackathon rules, not just a style note.
